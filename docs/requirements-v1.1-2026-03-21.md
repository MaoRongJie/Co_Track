# Co-Track 需求文档

- 版本：1.1
- 日期：2026-03-21

## 一、项目概述

### 1.1 项目背景
本系统是面向工业产品外观设计团队的分布式 AI 协作平台。首发场景是高铁外观涂层设计，后续可扩展到轨道交通车辆、汽车、消费电子、家电、机器人设备等。

### 1.2 核心目标
- 多设计师远程实时协作设计
- AI 辅助创意生成与图案绘制
- 工程参数自动估算（漆用量、工时、成本）
- 基于统一产品基准模型的三维实时外观预览
- 方案收集、分类、讨论、投票一体化
- 支持用户上传模型 / 模型库选择 / 自动生成近似基准模型三种 3D 基准模型来源

### 1.3 目标用户
- 会议主持人（项目负责人）
- 参与设计师
- 观察者（可选）

### 1.4 版本边界与非目标
- 当前不承担 CAD 主模型设计职责，不做参数化造型设计
- 每场会议仅允许一个统一基准模型
- 比较对象是外观方案，不是不同几何造型
- 自动生成模型仅作近似展示底模，不作为工程主模型
- 草图直接生成 3D 形体不进入本版主评审链路

## 二、系统功能需求

### 2.1 阶段 1：会议发起、目标设定与基准模型确认
- 主持人创建会议、生成邀请码，成员加入
- 输入自然语言设计目标
- 新增“产品与基准模型配置”区块：
  - 产品类别选择（高铁、城际、地铁、汽车、家电、其他工业产品）
  - 基准模型来源三选一：上传模型 / 模型库选择 / 自动生成近似模型
  - 自动生成时录入产品参数（轨交模板与通用工业模板）
- 系统解析设计目标，提取：主题、主色调、辅助色、风格关键词、特殊元素、限制提示
- 基准模型确认与锁定：
  - 上传/模型库：格式校验、归一化、UV 提取后确认
  - 自动生成：参数->参考提示/视图->3D API->归一化后确认
  - 主持人确认后锁定 `base_model_id`
  - 设计阶段统一使用同一 `base_model`
  - 生成未完成可临时用默认模板占位，评审前必须完成确认

输入：设计目标、产品类别、模型来源、文件或参数  
输出：`brief_json`、`product_profile_json`、会话级 `base_model` 元数据、广播简报与模型信息

### 2.2 阶段 2：个人方案绘制
- 每个成员独立 2D 画布，背景为当前会议基准模型 UV 模板图
- 同一会议内所有成员使用同一模板
- 画布能力：画笔、形状、填充、橡皮擦、撤销重做、颜色选择、图层管理（最多 5 层）、自动保存（30 秒）
- 辅助能力：分区叠加层、参考 3D 预览窗、基准模型来源标记
- AI 对话面板：创意建议、方案点评、元素推荐、图案生成、历史回溯
- 图案生成后可导入为参考层/底图层/贴花素材

### 2.3 阶段 3：方案收集与评审
- 主持人触发收集，上传画布截图，展示方案墙
- AI 语义聚类，主持人可手动调整
- 虚拟工程师分析：颜色面积、漆料用量、材料成本、工时、总成本、施工难度、置信度
- 修订：工程估算优先使用基准模型可喷涂面积 + UV 像素映射，不再依赖固定高铁校准系数
- 若来源为自动生成近似模型，报告标记“近似估算”
- 协作讨论：全局聊天（视频/语音/文字）、定向评论、1-5 星投票、实时排名
- 迭代支持：继续修改、保存新版本、AI 融合两个方案

### 2.4 阶段 4：三维预览与导出
- 加载会话级 `base_model`（GLTF/GLB 优先）
- 切换候选方案时，2D 画布导出 PNG 并实时映射到统一基准模型 UV
- 3D 交互：旋转、缩放、平移，光照预设（轨交/通用产品）
- 模型精度标签：Authoritative / Standard / Approximate
- 导出能力：
  - 始终支持：涂层展开图 PNG、工程报告 PDF、预览截图/多视角图
  - 条件支持：仅授权允许时导出带纹理 GLB
  - 授权受限时导出“预览包”（纹理图 + 视图图 + 报告）

## 三、Agent 设计

### 3.1 总览（6 个 Agent）
1. 会议协调 Agent（Orchestrator）
2. 目标解析 Agent
3. 产品模型 Agent（新增）
4. 创意助手 Agent
5. 图案生成 Agent
6. 方案分析 Agent

### 3.2 关键职责

#### 会议协调 Agent
- 会议创建后常驻，监听状态变更
- 工具：`broadcast_message`、`advance_stage`、`collect_designs`、`get_session_state`、`lock_base_model`
- 状态机：`LOBBY -> BRIEFING -> MODEL_PREPARING -> DESIGNING -> COLLECTING -> REVIEWING -> PREVIEWING`

#### 目标解析 Agent
- 输入：自然语言目标 + 产品类别
- 输出：结构化 Brief JSON（字段可缺省）
- Prompt：工业设计项目经理角色，输出主题、色彩、风格、元素、类别等

#### 产品模型 Agent（新增）
- 触发：阶段 1 选择模型来源后
- 职责：上传校验、模型库读取、参数生成近似模型、归一化、UV 提取、输出会话级 `base_model`
- 输出包括：`base_model_id`、`source_type`、`precision_level`、`model_url`、`uv_template_url`、`paint_regions`、`surface_area_m2`
- 工具：`validate_3d_file`、`normalize_model`、`extract_uv_template`、`generate_reference_views`、`build_generation_prompt`、`call_tripo_api`、`call_meshy_api`、`call_rodin_api`、`estimate_paintable_surface`、`save_model_asset`
- 说明：自动生成链路为“产品参数 -> 参考提示/视图 -> 3D API -> 归一化”，不假设一步式“参数 -> 精确 CAD”

#### 创意助手 Agent
- 上下文：Brief、产品类别、基准模型信息、最近 10 轮对话
- 模式：创意建议 / 方案点评 / 元素推荐

#### 图案生成 Agent
- 接收中文描述与可选参考图
- 转换 Prompt -> 调图像 API -> 返回 URL + Prompt
- 只负责 2D 外观图案素材，不负责几何重建

#### 方案分析 Agent
- 主持人收集方案后批量执行
- 子任务：聚类、工程估算、结果说明
- 输出：数据 JSON + 虚拟工程师点评 + 置信度标签（High/Medium/Low）
